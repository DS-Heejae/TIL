1. 프로젝트 개요

1-1. 주제

(1) 선정 배경

  • 다수의 화장품 추천 어플들이 피부 타입 등을 고려하지 않은 '총 평점'을 기준으로 제품 랭킹 리스트를 제공하고 있다. 개별 사용자의 나이, 피부 타입 등 특성에 따라 보여지는 랭킹 리스트도 달라져야 하지 않을까?

(2) 화장품 리뷰 플랫폼 '글로우픽'을 활용한 사용자 기반 화장품 추천

  • 글로우픽 URL
  • 사용자 정보 : 아이디, 성별, 연령대, 피부타입, 평점, 제품 리뷰

1-2. 뷰티 어플리케이션 시장분석 (2019.07 기준)

(1) 뷰티 어플 이용자 총 373만명.. 스마트폰 사용자 3,938만명 대비 약 9.5%

(2) 뷰티커머스 어플(화해, 글로우픽, 미미박스 등) 이용자 162만명

  • 뷰티 어플 5개 유형 중 가장 많은 이용자 (전체 대비 43%)- 올리브영, 랄라블라 등 H&B스토어 어플이 2위

(3) 뷰티커머스 어플, 10대~20대 사용 두드러져.. 40대 이상은 '올리브영'

  • 브랜드 충성도가 낮고, 다양한 제품들의 리뷰를 보고 합리적으로 소비하고자 하는 젊은 층 특성에 기인- 뷰티커머스를 제외한 타 유형에서는 30대 사용자가 가장 많음

(4) 오후 12시에서 14시, 18시에서 21시에 활성 사용자 수 '급증'

  • 젊은 여성들이 점심시간, 퇴근시간 활용해 어플에 접속하는 경향을 보임

1-3. 경쟁사분석 : 화해(화장품을 해석하다)

(1) 어플 출시: 화해(버드뷰) 2013년 vs 글로우픽(글로우데이즈) 2014년

(2) 매출 규모(2018년 기준) : 버드뷰 약 81억(+156%) vs 글로우데이즈 약 22억(+42%)

(3) 누적 리뷰수(2020.05.11 기준): 화해 511만256건 vs 글로우픽 348만8381건

(4) 핵심 경쟁력

  • 화해 : 성분 분석을 강조, 제품 정보 바로 하단에 성분에 대한 정보 상세 표기
  • 글로우픽 : 리뷰에 집중, 긍정리뷰와 부정리뷰 나눠보기 가능

2. 데이터 수집

(1) 각 카테고리별 100개 상품 정보 크롤링 (JSON)

  • 항목 : 브랜드명, 제품명, 제품아이디, 용량, 가격, 평점크롤링 코드

(2) 각 상품별 리뷰 크롤링 (JSON)

  • 항목 : 아이디, 연령, 피부타입, 성별, 평점, 리뷰(성별, 평점은 이미지 데이터)

    → 리뷰는 감성 분석으로 긍정/부정 단어 파악

3. 데이터 분석

3-1. EDA

(1) 데이터 전처리

ㄱ. 나이

  • 단위('세') 삭제
  • 100세 이상 이거나, 연도가 잘못 들어간 경우가 있어, 80세 이하만 선정

ㄴ. 가격:

  • 단위('원) 삭제,
  • '수퍼슬림 0.08g'처럼, 가격에 제품명과 단위가 들어간 경우 삭제

ㄷ. 사용자 id: 리뷰 개수가 2개 이상인 id만 선정

(2) 사용자 성별에 따른 나이 분포

  • 성별과 무관하게, 20대~30대초반 사용자가 주 고객임을 알 수 있다.
  • 특히, 남성의 경우, 20대 중반의 고객이 가장 많은 것을 알 수 있다.

(3) 사용자 피부 타입 분포

  • 본인 피부를 복합성으로 분류하는 사용자가 41.5%이고, 중성은 6.3% 정도만 나타나고 있다.
  • 복합성은 주로 얼굴의 중심 부위는 지성이고, 그 외 부분은 건조하거나 중성의 피부를 가지는 것을 의미하는데, 피부 타입 분류가 애매한 경우 복합성으로 생각하는 경우가 많아, 복합성의 비율이 높은 것으로 생각된다.
  • 피부 타입 정의(출처: 폴라초이스 뷰티정보

    • 중성 (유분기나 건조한 부위가 없는 피부)

    • 지성 (건조한 부위가 없으며 피부 전체로 유분기가 있음)

    • 건성 (각질이 일어날 수 있으며, 유분기가 전혀 없으며, 피부가 당기는 느낌을 가짐)

    • 복합성 (주로 얼굴의 중심 부위는 지성이며, 그 외부분은 건조하거나 중성의 피부를 가짐)

(4) 사용자 평점 분포

  • 전체 고객 중 4점을 준 고객이 가장 많았고, 그 다음은 5점, 3점 순서로 평점을 부여하였다.
  • 1, 2점을 준 고객이 많지 않은 것으로 보아, 3점은 긍정적인 평가를 한 것이 아님을 알 수 있다.
  • 이로 인해, 고객 리뷰 감성 분석 시, 데이터의 균형을 위해 1~3점은 부정, 4~5점을 긍정으로 라벨링하였다.

(5) 화장품 브랜드 분포

  • 이니스프리의 제품이 9.3%로 1위로 점유하고 있고, 미세한 차이로 에뛰드가 2위로 점유하고 있다.
  • 브랜드 점유율 별로 수치 상 큰 차이가 없어, 국내 화장품 시장의 경쟁이 치열함을 알 수 있다.

3-2. 고객 리뷰 감성 분석

(1) 피부타입별 리뷰 살펴보기 By 형태소 분석

  • 피부타입 : 건성(23.8%), 지성(14.1%), 민감성(14.1%), 중성(6.3%), 복합성(41.5%)
  • 건성 타입 : 본인 피부타입인 '건성'이라는 단어를 리뷰에 특히 많이 언급했다(총 8824번). 건조한 피부 특성 상 '보습'의 출현 빈도도 다른 피부타입 대비 높은 편이다.
  • 지성 타입 : '가격' 다음으로 '지속'이 많이 언급된 점이 특징이다(총 4479번). 유분이 많아 화장이 금방 번지는 피부 특성 상 화장품의 지속력을 중요시하는 것으로 보인다. 건성 타입 주요 단어에 없던 '여드름'이 총 3022번 나왔고 '트러블'의 빈도도 중위권에 속한다. 유분으로 인한 여드름 등 피부 트러블이 지성 타입 소비자들의 고민 사항임을 유추할 수 있다. 또한 이들은 '매트'한 '파우더' 타입의 화장품을 선호하는 것으로 보인다.
  • 민감성 : '트러블'이라는 단어가 총 4547번으로 '향', '가격' 다음 많이 출현했다. '성분', '민감', '진정' 등도 특징적인 단어다. 자극에 민감하고 트러블이 쉽게 발생하는 피부 특성 상 '성분'을 중요시하며 '피부 진정'에 비교적 관심이 많은 것으로 보인다.
  • 중성 : 위 3가지 피부 타입에 언급된 단어들이 고르게 나오며 특징적인 단어는 눈에 띄지 않는다. 복합성 타입도 중성과 결과가 비슷해 생략한다.
  • 건성, 지성, 민감성, 중성 모두 '가격'과 '향'이 출현 빈도 TOP 3 단어에 속한다. 피부 타입에 관계 없이 화장품 소비자들이 제품을 선택할 때 중요시하는 요소로 해석할 수 있다.

(2) 감성 분석으로 긍정/부정 리뷰 판별하기

  • 소비자들은 화장품 리뷰를 남길 때 1~5점 사이의 평점(score)을 함께 매길 수 있다. 1~3점은 부정적인 반응(negative), 4~5점은 긍정적인 반응(positive)으로 구분했다.
  • 긍정적인 반응은 1, 부정적인 반응은 0으로 라벨링해 새로운 Column인 'po_ne'를 생성했다.

  • 학습(Train) : 7만개 리뷰를 한글 형태소 분석기인 KoNLPy를 사용해 형태소 단위로 토큰화했다. 여기에 TF-IDF Vectorizer를 적용해 문서 구별 능력에 따라 단어에 가중치를 뒀다. 벡터로 변환한 리뷰에 LightBGM Classifier를 사용, 긍정/부정으로 분류하는 학습을 시켰다.
  • 테스트(Test) : 나머지 3만개 데이터로 테스트를 진행한 결과 Accuracy 84.7%를 얻었다.

from sklearn.metrics import accuracy_score

tfidf_matrix_test = tfidf_vect.transform(test_df['review'])
preds = lgbm_clf.predict(tfidf_matrix_test)

accuracy_score(test_df['po_ne'], preds)

4. 추천 시스템

4-1. 협업 필터링 개념

: 많은 사용자들로부터 얻은 기호 정보에 따라, 사용자들의 관심사들을 자동적으로 예측하게 해주는 방법

(1) 사용자 기반 협업 필터링

나와 비슷한 성향을 지닌 사용자를 기반으로, 그 사람이 구매한 상품을 추천

(2) 아이템 기반 협업 필터링

내가 이전에 구매했던 아이템을 바탕으로, 그 상품과 유사한 다른 상품을 추천

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import linear_kernel

tf1 = TfidfVectorizer(analyzer='word', ngram_range=(1, 2),
                      min_df=0)
tfidf_matrix1 = tf1.fit_transform(item_dfs['tag'])
cosine_sim1 = linear_kernel(tfidf_matrix1, tfidf_matrix1)

titles = item_dfs['NAME']
indices = pd.Series(item_dfs.index, index=item_dfs['NAME'])
def tags_recommendation(product_name):

    product_name = input('제품명을 입력하세요 : ')

    print('\n')
    print('선택한 제품의 정보 : ')

    display(item_dfs[item_dfs['NAME'] == product_name])

    idx = indices[product_name]
    sim_scores = list(enumerate(cosine_sim1[idx]))
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)
    sim_scores = sim_scores[1:11]
    cosmetics_indices = [i[0] for i in sim_scores]

    print('\n')
    print('추천 화장품 : ')

    recommend = pd.DataFrame(titles.iloc[cosmetics_indices])
    recommend['sim'] = sim_scores
    recommend = recommend.merge(
        item_dfs[['NAME', 'BRAND', 'keywords']], left_on='NAME', right_on='NAME', how='inner')

    return recommend


reco = tags_recommendation(product_name)

reco['similarity'] = [r for i in range(
    len(reco)) for r in reco['sim'][i]][1::2]

5-2. 기대 효과

(1) 판매자 측면

  • 대량 맞춤을 통하여 일대일 마케팅이 가능, 웹 개인화 맞춤화 서비스가 가능하여 고객의 충성도 높일 수 있음.

(2) 구매자 측면

  • 사용자의 관심을 기반으로 하여, 상품 선택 시 탐색 비용을 감소시켜줌(정보 과부하 현상 완화)
  • 새로운 상품을 발견하는 즐거움을 바탕으로, 추가적인 제품, 서비스 사용을 유도
  • 다른 사용자들이 더 좋게 평가한 제품을 추천함으로써 검증된 아이템을 먼저 제시 받을 수 있음.

6. 프로젝트 회고

6-1. 협업 필터링 방식의 한계

(1) 콜드 스타트

  • 평점을 매기지 않은 신규 사용자에게 화장품 추천 불가.

(2) 계산 효율 저하

  • 사용자가 일정 수준 이상 많아야 정확한 결과를 낼 수 있으나, 협업 필터링 자체가 계산량이 많은 알고리즘이라서, 사용자가 많아질수록 시간 효율성이 떨어질 수 있음.

(3) 롱테일

  • 롱테일: 파레토 법칙(전체 결과의 80%가 전체 원인의 20%에서 일어나는 현상)을 그래프로 나타내었을 때, 꼬리처럼 긴 부분을 형성하는 80%의 부분을 일컫는 말.
  • 사용자들의 관심이 저조한 화장품은 정보가 부족하여 추천되지 못하는 문제 발생.
